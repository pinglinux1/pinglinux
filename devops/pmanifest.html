Puppet Manifest

Share

What is manifest?

Puppet programs are called manifests, and they use the .pp file extension. They are mainly written in the manifest and modules directory of puppet master. Manifest are written in DSL.

Puppet Manifest example:

it is written the the master server /etc/puppet/manifest/site.pp

Manifest written to install httpd package and to run it 

#cat /etc/puppet/manifest/site.pp
		  package {'httpd':  
		         ensure => installed,
		}
		service {'httpd':
		  ensure => running,
		}

Install httpd package. If installed already, do nothing

Start the httpd service. If its running, do nothing

 # Manifest to override the file, change its permission and to restart the service.
file {'ssh-config':
    ensure => file, 			  # making sure it is the file
    path    =>/etc/ssh/sshd_config, 	   # it is the path where file
    mode => 600,                           #changing the file permission
    replace => no,     #dont override if the file already exists.
    source =   '/root/examples/sshd_config',  # File which will replace the sshd_config file
}
    


service {'sshd',
         	ensure => running,  # same as  service sshd start
        	enable => true,       # same as chkconfig sshd on
        	subscribe => File['/etc/ssh/sshd_config'], 
		# whenever there is change on this file, 
		#restart the sshd service
}

noop (no operation )

This is useful for seeing what changes Puppet will make without actually executing the changes.This gets handy in the test purpose to make sure your code doesnt throw error at you.

puppet agent -t --noop 
puppet apply -noop /modules/httpd/manifest/init.pp
