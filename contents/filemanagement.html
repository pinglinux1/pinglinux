What is file management ?

The system that an operating system uses to organize and keep track of files.Every thing in Linux is file. It is very important to have the solid grasp of linux basic commands to manage the files.Lets get started.Connect to your Linux server Remotely using SSH and continue below.

1) Finding your current location in the system

pwd (Present working directory)

sh-3.2# pwd
/Users/Biks/Desktop
sh-3.2#

2) Changing the location or directory back and forth

cd ../ (change directory)
cd dirname/ 
			sh-3.2# pwd
			/Users/Biks/Desktop
			sh-3.2# cd ../
			sh-3.2# pwd
			/Users/Biks
			sh-3.2# cd Desktop/
			sh-3.2# pwd
			/Users/Biks/Desktop
			sh-3.2#

3) Making the directory

mkdir (make directory) 
             		sh-3.2# ls
			sh-3.2# mkdir newdirectory
			sh-3.2# ls
			newdirectory
			sh-3.2#

4) Renameing the directory

mv (move) 
                        sh-3.2# ls
			newdirectory
			sh-3.2# # #we will rename the newdirctory to olddirectory
			sh-3.2# mv newdirectory olddirectory
			sh-3.2# ls
			olddirectory
			sh-3.2#

5) Delete the directory

rm -f (remove)
 				sh-3.2# ls
				olddirectory
				sh-3.2# rm -rf olddirectory/
				sh-3.2# ls
				sh-3.2#

6) Move the directory or files

mv 
                         sh-3.2# ls
			 sh-3.2# mkdir pinglinux
			 sh-3.2# ls
			 pinglinux
			 sh-3.2# mv pinglinux/ ../
			 sh-3.2# ls
			 sh-3.2#

7) Creating the files

touch filename 
        				sh-3.2# ls
					sh-3.2# touch file1 file2 file3
					sh-3.2# touch file4
					sh-3.2# ls
					file1	file2	file3	file4
					sh-3.2#

8) Creating the files.Vi lets you add the contents to the file, where as touch only creates the file.

vi filename 
        			sh-3.2# vi file5
				sh-3.2# ls
				file5
				sh-3.2#

9) Listing the files and directories in current location

ls (listing)
ls -a shows the hidden files as well
ls -l shows the long listing of the file with information.
                         sh-3.2# ls
			file5
			sh-3.2# ls -a
			.	..	file5
			sh-3.2# ls -l
			total 8
			-rw-r--r--  1 root  staff  4 Dec 20 16:21 file5
			sh-3.2#

10) Display the contents of the files

cat filename 
                        sh-3.2# vi file1 # and Inserted a b c d e f 
			sh-3.2# cat file1
			a
			b
			c
			d
			e
			f
			sh-3.2#

11) Displaying the contents from top of the file

head 
       			 sh-3.2# head -3 file1
			 a
			 b
			 c
			 sh-3.2#

12) Displaying the contents from bottom of the file

tail 
                        sh-3.2# tail -3 file1
			d
			e
			f
			sh-3.2#

13) View a text file one page at a time.Press spacebar to go to next page

more 
  		          sh-3.2# more filename

14) View a text file one page at a time.And allows to navigate up and down which is not possible with more command

less 
			sh-3.2# less filename

15) Counting the number of lines and word in the file

wc (word count)
			 sh-3.2# wc file1
      			 6       6      12 file1
			 sh-3.2#

16) copying the file and directory

cp (copy)
                          sh-3.2# ls
			  file1
			  sh-3.2# cp file1 file2
			  sh-3.2# ls
			  file1 file2
			  sh-3.2#

17) Sends the output of one command to another one for further processing

| pipe
			 sh-3.2# cat file1 | grep a
			 a
			 sh-3.2# # the result was a b/c file contains letter a
                        sh-3.2#

18) Searches the contents of files or commands that matches certain criteria

grep 
			 sh-3.2# cat file1 | grep a 
                         a
                         sh-3.2# # the result was a b/c file contains letter a
                        sh-3.2#

16) Creating the link of the file and directories

ln (link)
                          sh-3.2# ls
                          file1
                          sh-3.2# ln -s file1 softlinkfile
                          sh-3.2# ln file1 hardlinkfile
                          sh-3.2# ls -l
                          total 24
                          -rw-r--r--  2 root  staff  12 Dec 20 16:32 file1
                          -rw-r--r--  2 root  staff  12 Dec 20 16:32 hardlinkfile
                          lrwxr-xr-x  1 root  staff   5 Dec 20 16:34 softlinkfile -> file1
                          sh-3.2#
           
 Note:Following are the differeces between hardlink and soft links:
Hard links have same inode numbers while softlinks do not 
Hard links have actual file contents but softlinks are just pathway 
Removing hard link just affects one file but removing original file affects the softlink. 
Hard links cannot be created for the directory but softlinks can be created for directory.
