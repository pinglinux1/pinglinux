Git Commands

Basic Git commands

init,clone,add, commit,log, push, status

git init

Initializing the local directory as a Git repository. 

[root@puppetmaster html]# git init
Initialized empty Git repository in /var/www/html/.git/

git clone {repo url}

Clones the repository into current directory

[root@puppetmaster html]# git clone https://github.com/pinglinux0/hello_world.git
Initialized empty Git repository in /var/www/html/hello_world/.git/
remote: Counting objects: 9, done.
remote: Compressing objects: 100% (5/5), done.
remote: Total 9 (delta 2), reused 8 (delta 1), pack-reused 0
Unpacking objects: 100% (9/9), done.

git add

Adds the file to be committed. 

 git add . (adds all the file) 
git add filename (adds individual file

git commit

Saves the commited stage with the Messages

git commit -m "new file added"

git status

shows the file that has been modified but hasn't been committed 

git status
[root@puppetmaster html]# ls
hello_world  index.html  index.php  newfile  README.md
[root@puppetmaster html]# echo "lets modify new file" > newfile 
[root@puppetmaster html]# git status
# On branch master
# Changed but not updated:
#   (use "git add ..." to update what will be committed)
#   (use "git checkout -- ..." to discard changes in working directory)
#
#	modified:   newfile
#
no changes added to commit (use "git add" and/or "git commit -a")

git log

Shows all the commited history 

[root@puppetmaster html]# git log

Author: root 
Date:   Sun Jan 17 22:22:00 2016 -0500

    new file added

Roll back commited history back and forth

git revert commit-id

Revert back to previous stage temporarily 

 git log

Displays all the committed history. To roll back, pick the stage you want to roll back and do git revert commit-id for that particular stage 

git revert 77696af8842c6e890823f843cd9af06ebbbef884

And to roll back to the head again, since this is temporary roll back, you can do git log and pick the head commit id and roll it back

git checkout commit-id

You can use git checkout to fetch an old version of single file.

git checkout commit-id filename
 git checkout master

will rollback to the most recent statge

git revert and git checkout

git checkout can be used to checkout branch or specific commit in your local working copy and this will not make any changes to the history where as git revert will add new commit, i.e changes the history.

Branching and Merging

Branching

it is mainly new working directory or staging area.New commits will be saved in the history of current branch

List all the branches 

 git branch

Creates new branch named branchname

 git checkout -b branchname

Switch branches

 git checkout branchname

Deletes branches

 git branch -d branchname

Merging

Process of putting back the history of new branch to the old master branch 

 git merge branchname

Example:
# create new branch
git checkout -b newbranchname

#modifies some file
git add
git commit -m "modified file"

#modifies some file again
git add
git commit -m "modified file again"

#Merge the branch
git checkout master
git merge branchname
git branch -d branchname
